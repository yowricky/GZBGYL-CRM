# 客户关系管理平台 PRD

版本：v0.1  
日期：2026-06-24  
项目：葛洲坝集团供应链管理有限公司客户关系管理平台

## 1. 产品背景

本项目面向系统集成、软件企业及供应链协同业务场景，建设一套企业级 CRM 平台，用于统一管理客户、合作伙伴、线索、商机、项目跟进、报价、合同、回款、待办活动、经营报表和系统权限。

平台当前以企业内部私有化部署为目标，服务对象包括销售、销售主管、项目经理、运营管理、财务查看、合作伙伴协同人员、高级管理人员和系统管理员。

## 2. 产品目标

1. 建立统一客户与合作伙伴档案，沉淀组织关系、联系人、合作历史和项目记录。
2. 将商机与项目分离：商机侧重线索培育和机会判断，项目侧重立项后的跟进过程管理。
3. 支持项目型销售闭环：线索、客户、商机、项目、报价、合同、回款、活动、报表。
4. 将工作台建设为经营入口，展示年、季、月团队及个人业绩要求和完成情况。
5. 为不同主体模块提供统一的“关键指标、搜索筛选、主体清单、详情/操作”页面结构。
6. 建立 Admin 系统管理入口，将组织架构、用户、角色、权限等全局设置归拢到 Admin 菜单。
7. 支持未正式上线前的便捷测试登录和无密码创建用户。
8. 形成简洁、清爽、专业、企业级的 UI 体验，整体参考 Salesforce 的对象式业务组织方式。

## 3. 用户角色

| 角色 | 职责 | 权限重点 |
| --- | --- | --- |
| 销售 | 维护本人客户、线索、商机、项目跟进和活动 | 查看和处理本人负责数据 |
| 销售主管 | 管理团队线索、客户、商机和经营情况 | 查看部门及下级团队数据 |
| 项目经理 | 参与项目过程、报价、合同、回款协同 | 查看分配项目及相关业务信息 |
| 运营管理 | 查看项目进度、经营指标、过程风险 | 查看授权部门经营数据 |
| 财务查看 | 查看合同、回款、财务相关字段 | 查看授权合同和回款信息 |
| 合作伙伴 | 参与线索推荐、联合项目和协同活动 | 查看授权合作业务信息 |
| 高级管理 | 查看经营驾驶、报表、关键风险 | 查看公司级汇总数据 |
| 系统管理员 | 管理组织、用户、角色、权限和全局设置 | 管理系统配置，不承载业务模块菜单 |

## 4. 信息架构

### 4.1 顶部主体菜单

平台菜单围绕业务主体组织，而不是按孤立功能堆叠。

| 一级主体 | 二级能力 |
| --- | --- |
| 工作台 | 经营目标、团队/个人业绩、今日处理、商机阶段 |
| 报表分析 | 经营指标、客户增长、阶段分析、合同回款 |
| 客户 | 客户档案、联系人、待办与活动、报价、合同与回款 |
| 合作伙伴 | 伙伴档案、伙伴线索、联合项目、待办与活动、报价、合同与回款 |
| 商机 | 线索管理、商机池、转项目判断、待办与活动 |
| 项目 | 项目跟进、行动计划、报价、合同与回款、待办与活动 |
| Admin | 组织架构、用户管理、角色权限 |

### 4.2 Admin 菜单边界

Admin 仅承载全局性设置，不展示客户、合作伙伴、商机、项目等业务模块内容。

Admin 当前包含：

- 组织架构
- 用户管理
- 角色权限

## 5. 核心业务模块

### 5.1 工作台

工作台用于展示经营和个人工作状态，不重复出现“工作台”大标题，不出现装饰性大字。

核心内容：

- 年度团队目标完成情况
- 季度团队目标完成情况
- 个人月度目标完成情况
- 今日必须处理事项
- 商机阶段分布

商机阶段及赢率：

| 阶段 | 赢率 |
| --- | ---: |
| 初步接触 | 0% |
| 方案交流 | 40% |
| 立项/预算 | 50% |
| 投标/报价 | 60% |
| 商务谈判 | 85% |

### 5.2 报表分析

报表分析用于查看经营结果和趋势。

页面结构：

- 顶部关键指标
- 搜索、状态、日期选择并列摆放
- 下方展示报表主体清单
- 支持按周期、负责人、状态、金额等维度筛选

### 5.3 客户模块

客户模块用于维护客户档案、联系人、合作历史和关联业务。

页面结构：

- 关键指标在最上方
- 搜索栏在关键指标下方
- 主体列表展示当前用户跟进的客户
- 主体列表右侧提供“详情”“删除”按钮
- 顶部保留“新建客户”按钮

客户新建要求：

- 新建客户时必须录入客户基础信息的所有字段
- 增加组织机构代码
- 输入框排列应尽量避免左右拖动和上下拖动
- 客户维护信息参考《项目情况登记表.xlsx》

客户维护信息分组：

- 客户基础信息
- 关联项目情况
- 销售地图
- 客户关系与态度
- 行动计划与执行

### 5.4 合作伙伴模块

合作伙伴模块参考客户模块和 Salesforce 对象页设计。

核心能力：

- 合作伙伴档案
- 推荐线索
- 联合项目
- 协同活动
- 报价
- 合同与回款

列表中如已存在“新建”入口，主体列表操作按钮使用“详情”，不再重复出现“增加”。

### 5.5 商机模块

商机与项目分离。

商机重点：

- 线索管理
- 客户意向识别
- 需求确认
- 预算和决策链判断
- 是否转入项目

商机不承担完整项目过程管理。

### 5.6 项目模块

项目重点是立项后的跟进过程管理。

核心能力：

- 项目进度
- 行动计划
- 执行成本
- 报价协同
- 合同协同
- 回款协同
- 项目复盘

项目经理、运营管理、财务查看可共同查看关键字段和协同事项。

### 5.7 报价、合同与回款

报价：

- 支持报价记录、状态、金额、关联客户/项目
- 支持后续扩展报价版本和附件

合同与回款：

- 合同金额、签署状态、履约状态
- 回款计划、实际回款、回款风险
- 财务查看角色可查看授权合同与回款信息

### 5.8 待办与活动

活动用于记录与客户、合作伙伴、商机、项目相关的任务、会议、电话、报价确认、回款复核等事项。

## 6. 全局页面布局规范

除工作台外，所有功能模块采用统一布局：

1. 顶部：关键指标。
2. 中部：搜索、状态、日期选择并列摆放。
3. 下方：主体清单，根据搜索条件筛选。
4. 右侧操作：主体列表右侧显示“详情”“删除”等操作。
5. 左侧浮动栏：处理重点，展示与中间栏选中主体相关的信息汇总或提示，不放表格数据。
6. 右侧浮动栏：AI 项目助手，与左侧栏同宽，用于交互和建议。

布局要求：

- 中间栏为主内容区域。
- 左右栏浮动跟随。
- 中间栏内容不能被截断。
- 所有标题不能换行，空间不足时使用省略号。
- 删除每个内容框中的说明文字，保留标题和业务数据。

## 7. UI 与视觉规范

设计目标：

- 简洁
- 清爽
- 专业
- 有商业设计感
- 不过度暗色
- 不像普通系统模板

品牌方向：

- 颜色以葛洲坝集团供应链管理有限公司 Logo 主题为准。
- 主色使用企业蓝。
- 风险色使用受控红色。
- 页面以白色和浅灰为主要背景。

字体规范：

- 标题：微软雅黑
- 正文：宋体
- 所有数字和英文：Times New Roman
- 除标题和功能模块名外，其他字体不加粗
- 功能模块名加粗

组件规范：

- 圆角克制
- 间距稳定
- 卡片轻边框、低阴影
- 按钮清晰但不花哨
- 导航栏围绕业务主体组织

## 8. 登录与测试阶段账号策略

当前为测试阶段：

- 登录仅需用户名
- 暂不要求输入密码
- 新建用户不要求设置密码
- 默认测试账号为 `admin`

正式上线前应恢复密码、会话安全和登录失败保护策略。

## 9. 角色与权限

角色管理要求：

- 用户可分配多个角色
- 每个角色的权限可配置
- 角色名称在 Admin 页面全部使用中文
- 权限名称在角色权限页面全部使用中文
- 权限编码仅作为系统内部提交值，不在界面中直接展示

当前内置角色：

- 销售
- 销售主管
- 售前技术
- 项目经理
- 运营管理
- 财务查看
- 高级管理
- 系统管理员

权限示例：

- 系统管理
- 查看本人商机
- 查看部门商机
- 查看分配项目
- 查看授权合同
- 查看授权回款
- 查看部门财务字段
- 新建客户
- 查看客户
- 编辑客户
- 删除客户

## 10. 数据模型概览

主要业务对象：

- Lead：线索
- Account：客户
- Contact：联系人
- Opportunity：商机
- Project：项目
- Quote：报价
- Contract：合同
- Payment：回款
- Activity：活动
- OrganizationUnit：组织架构
- User：用户
- Role：角色
- Permission：权限

数据通用要求：

- 主要业务对象使用 UUID 主键
- 关键实体支持版本号
- 关键变更应写入审计日志
- 业务对象支持附件扩展

## 11. 非功能需求

部署：

- 使用 Docker Compose 本地和私有化部署
- PostgreSQL 存储业务数据
- Redis 用于会话和短期缓存
- MinIO 或兼容 S3 的对象存储用于附件
- Nginx 提供前端静态资源和反向代理

性能：

- 支持 50-300 名企业用户
- 常规列表、详情和保存接口目标响应时间 P95 小于 2 秒
- 大数据导入导出应走后台任务

安全：

- 正式上线应启用密码校验
- 权限变更、用户停用、角色调整应记录原因
- Admin 权限与业务权限分离
- 敏感字段按角色和授权范围查看

## 13. 验收标准

1. 用户可通过本地地址访问系统并使用 `admin` 登录。
2. 顶部显示“客户关系管理平台”。
3. 工作台展示团队及个人业绩要求和完成情况。
4. 客户、合作伙伴、商机、项目等模块点击后可进入对应页面。
5. 除工作台外，各模块均遵循“关键指标、搜索筛选、主体列表”的统一布局。
6. 主体列表右侧操作不被截断。
7. 左右浮动栏跟随页面，中间栏完整显示。
8. Admin 仅展示系统管理项，不展示业务模块内容。
9. 用户管理中可创建用户、分配多个角色。
10. 角色权限页面可查看角色、勾选权限并保存。
11. 角色和权限在界面中均显示中文。
12. 标题不换行，内容框说明文字已清理。
13. Docker 前后端服务健康。
14. 前端测试与生产构建通过。
