# 企业级项目型 CRM 系统设计

## 1. 背景与目标

本系统面向系统集成和软件企业，以项目型直销为核心，服务单一企业内部的多个部门及分公司。首版打通以下销售闭环：

线索 -> 客户与联系人 -> 商机 -> 报价 -> 合同 -> 回款

系统部署在企业自有服务器或专属云环境，预计服务 50-300 名用户。桌面 Web 承担完整业务管理，手机自适应 Web 承担查询、待办、跟进和行动执行。

### 1.1 首版目标

- 建立统一、可查重的客户和联系人主数据。
- 规范线索分配、转化和项目型商机推进。
- 将现有《项目情况登记表》中的项目作战方法结构化、在线化。
- 支持产品/服务目录与自由报价项并存，保留报价版本。
- 管理合同、回款计划、实际回款和逾期风险。
- 提供年、季、月团队及个人业绩目标与完成情况。
- 实现本人、部门层级和显式协作相结合的数据权限。
- 通过 Excel 导入导出和标准 REST API 为后续集成留出边界。

### 1.2 首版非目标

- 不建设 SaaS 多租户能力。
- 不开发原生移动 App。
- 不建设报价或合同审批流，审批在线下完成后回填结果。
- 不在首版连接 ERP、财务、邮箱、日历、电子签章或企业通讯平台。
- 不采用微服务，不建设复杂分布式基础设施。
- 不覆盖采购、实施交付、售后工单等完整 ERP/项目管理流程。

## 2. 已确认的产品原则

- 单企业、私有化部署。
- 项目型直销，标准销售阶段。
- 销售数据按“本人 + 部门及下级部门”可见。
- 商机由一名销售负责人负责，可邀请售前、技术、项目经理等协作。
- 自行录入的线索归创建人；公司统一导入、市场或管理层录入的线索由主管分配；首版无公海回收。
- 客户按企业名称和统一社会信用代码查重；重复客户不能直接创建，归属冲突由主管处理。
- 报价、合同采用结构化记录与附件，不在线生成 Word/PDF。
- 合同同时管理回款计划和实际回款。
- 产品目录与一次性自由项并存。
- 销售预测同时提供阶段加权金额和“潜在/较大把握/承诺”分级口径。

## 3. 总体架构

采用模块化单体架构：

- 前端：Vue 3、TypeScript、企业级组件库、响应式布局。
- 后端：Java、Spring Boot 3、Spring Security、模块化领域边界。
- 数据库：PostgreSQL。
- 缓存：Redis，仅用于会话、短期缓存和限流辅助，不作为业务事实来源。
- 文件：企业对象存储或兼容 S3 的私有文件服务，数据库仅保存元数据。
- 入口：Nginx，负责 HTTPS、静态资源和反向代理。
- 部署：Docker Compose，支持配置外置、数据卷、备份和版本回滚。
- 接口：前后端使用 REST API；对外提供受控的标准 API。

模块只能通过公开应用服务协作，禁止跨模块直接修改数据。首版作为单一部署单元运行，但模块保留独立领域模型和接口，以便未来按业务压力拆分。

### 3.1 业务模块

1. 组织、用户、角色与数据权限
2. 线索
3. 客户、客户关系与联系人
4. 商机与销售预测
5. 项目跟进作战
6. 产品/服务目录与报价
7. 合同与回款
8. 活动、待办与提醒
9. 业绩目标与经营看板
10. 附件
11. Excel 导入导出
12. 审计与系统配置

## 4. 核心业务模型

### 4.1 组织与用户

- `OrganizationUnit`：公司、部门、下级部门，形成树形层级。
- `User`：员工账号，属于一个主要部门，可拥有多个角色。
- `Role`、`Permission`：功能权限、数据范围和敏感字段权限。

### 4.2 线索

- `Lead`：企业/联系人线索、来源、负责人、状态和最近跟进。
- 状态：待分配、待联系、跟进中、已转化、无效。
- 转化时先查重，再创建或复用客户，创建联系人和商机，并安排首次行动。
- 线索转化是单个事务；任何步骤失败时整体回滚。

### 4.3 客户与关系网络

- `Account`：企业客户主档，包含名称、信用代码、行业、区域、官网、企业规模和归属。
- `AccountRelation`：集团/子公司、合作伙伴、关联企业等客户间关系。
- `Contact`：联系人、部门、职务、项目角色、影响力和联系方式。
- `OpportunityParty`：客户在具体商机中的角色，如最终用户、招标方、总包方、合作伙伴、我方主体。

客户主档关系与商机参与方分开：同一客户在不同商机中可以扮演不同角色。

### 4.4 商机

- `Opportunity`：项目名称、负责人、阶段、预计金额、预计签约月、赢率、承诺等级和状态。
- `OpportunityTeamMember`：销售负责人、售前、技术、项目经理及其他协作成员。
- `OpportunityStageHistory`：阶段变化、回退原因、操作者和时间。
- 状态：进行中、暂缓、赢单、输单。
- 加权金额 = 预计金额 x 当前阶段赢率。

### 4.5 产品、报价、合同与回款

- `ProductService`：软件、硬件、实施、运维等目录项，包含标准价、成本价和税率。
- `Quote`、`QuoteVersion`、`QuoteLine`：报价及不可覆盖的历史版本；明细可引用目录项，也可为一次性自由项。
- `Contract`：合同编号、签约主体、金额、签约日期、状态和附件。
- `PaymentPlan`：回款节点、计划日期和计划金额。
- `PaymentReceipt`：实际回款日期、金额和凭证；错误记录通过更正/作废处理。

## 5. 项目跟进作战模型

本节依据用户提供的《项目情况登记表.xlsx》设计。商机拥有统一的项目作战页，原表中的重复信息由主数据和关联对象自动带入。

### 5.1 项目驾驶舱

- 客户、销售经理、售前支持、项目类型和预计签约时间。
- 项目需求、必上理由、项目预算、预算日期和预算清单。
- 当前销售阶段、赢率、承诺等级、加权金额和最新动态。
- 下一行动、重大风险、里程碑延误和最后更新时间。

### 5.2 项目进度与决策链

- `ProjectMilestone`：可研、立项、资金批复、招采、签约、到货、验收等。
- 每个里程碑记录计划日期、实际日期、状态和延误原因。
- 不同项目类型可使用不同里程碑模板，也允许项目内调整。
- `DecisionStep`：决策步骤、负责组织、关键条件、顺序和当前所在环节。
- 项目里程碑与销售阶段独立；系统只给出调整建议，不自动改变销售阶段。

### 5.3 项目组织和销售地图

- 项目小组成员关联所属参与方、职位、项目角色和影响力。
- “知彼”记录竞争对手的销售/方案人员、方案、优势、弱势、关系、策略和潜在危险。
- “知己”记录我方与合作伙伴的方案、优势、弱势、客户关系和协同策略。
- “知客户”记录关键人的支持/中立/反对态度、职位、角色、原因、策略和效果。
- 关键人态度、预算、预计签约金额和竞争风险的变化保留历史。

### 5.4 行动计划、执行和复盘

- `ActionPlan`：行动目标、负责人、计划起止时间、关联里程碑和状态。
- `ActionParticipant`：我方、合作伙伴、客户或其他资源及具体参与人。
- `ActionExecution`：实际起止时间、执行结果、效果、反思和下一步。
- `ActionCost`：差旅、人工、办公及杂项、其他成本，自动汇总。
- 行动逾期、长期未跟进和下一行动缺失时产生提醒。
- 跟进、字段变更、附件、执行和复盘按时间形成统一时间线。

## 6. 商机阶段与门禁

| 阶段 | 默认赢率 | 核心进入条件 |
| --- | ---: | --- |
| 初步接触 | 0% | 客户、项目名称、负责人、来源、预计金额或区间、预计签约月、下一行动 |
| 需求确认 | 20% | 项目需求、项目类型、至少一名关键联系人 |
| 方案交流 | 40% | 售前/技术协作人、方案摘要、客户反馈 |
| 立项/预算确认 | 50% | 预算状态、关键里程碑、当前决策环节 |
| 投标/报价 | 60% | 有效报价版本或投标信息、产品/服务明细、预计投标日期 |
| 商务谈判 | 85% | 谈判金额、预计签约日、商务条件摘要 |
| 赢单 | 100% | 合同、签约金额、签约日期、回款计划 |
| 输单 | 0% | 输单原因、竞争对手、复盘结论 |

阶段门禁采用混合策略：核心字段强制，作战分析信息给出完整度提示。阶段回退、人工覆盖赢率、金额或预计签约日期变化、赢单/输单重新开启均需填写原因并审计。

## 7. 角色、权限和数据范围

### 7.1 角色

- 销售：管理本人负责的线索、客户和商机。
- 销售经理：查看本部门及下级部门，分配线索、处理归属冲突、转移客户/商机和查看预测。
- 售前/技术：查看被邀请参与的商机，维护方案、技术里程碑、行动和附件；默认看不到成本、毛利和回款明细。
- 项目经理：只读查看被关联项目的销售、报价、合同和回款信息。
- 运营管理：只读查看授权部门或全公司的经营数据。
- 财务查看：只读查看合同、回款计划和实际回款。
- 管理层：全公司只读，查看经营看板、预测、漏斗、重点项目和风险。
- 系统管理员：管理组织、用户、角色、配置、模板和导入，不默认拥有业务数据修改权。

### 7.2 数据权限

- 默认范围为本人负责的数据。
- 销售经理继承本部门及下级部门的数据范围。
- 商机团队成员通过显式授权获得相关商机及必要客户信息；移出团队后立即收回。
- 全库客户查重不受部门隔离影响，但无权限用户只能看到企业名称、区域、归属部门和负责人，不能打开详情。
- 成本、毛利、合同、回款和联系人联系方式使用字段级权限控制。

## 8. 页面与操作路径

### 8.1 主导航

- 工作台
- 线索
- 客户与联系人
- 商机
- 报价
- 合同与回款
- 待办与活动
- 报表分析
- 系统管理

顶部提供全局搜索、快速新建、提醒和个人菜单。

### 8.2 商机作战页

采用以下页签：总览、项目进度、组织与关键人、销售地图、行动计划、报价、合同与回款、时间线。

### 8.3 核心操作路径

1. 线索转化：查重 -> 关联/创建客户 -> 创建联系人 -> 创建商机 -> 安排首次行动。
2. 商机推进：记录跟进 -> 更新作战页 -> 完成阶段门禁 -> 调整预测 -> 生成下一行动。
3. 报价签约：选择目录/自由项 -> 保存版本 -> 标记客户确认版本 -> 建立合同和回款计划。
4. 回款闭环：节点提醒 -> 登记实收 -> 更新回款率 -> 逾期预警或更正留痕。

### 8.4 手机端范围

首版支持我的待办、客户/商机查询、快速跟进、拍照上传附件和更新行动执行情况。复杂报价编辑、批量导入、权限配置和大型报表仅在桌面端提供。

## 9. 工作台、业绩目标和报表

工作台按用户权限显示：

- 年度、季度、月度切换。
- 团队、个人维度切换。
- 签约目标、实际签约、完成率和差额。
- 回款目标、实际回款、完成率和差额。
- 加权商机储备、目标覆盖倍数、承诺商机金额和目标缺口。
- 目标完成趋势和团队成员排名。
- 商机漏斗、阶段转化率、重点商机和风险。
- 今日待办、逾期行动、长期未跟进和回款风险。

运营管理或授权管理员按年度维护部门和个人签约额、回款额目标。季度和月度目标可手工分解；未分解时仅展示年度累计进度。签约完成额来自已生效合同，回款完成额来自已确认实际回款。团队指标按组织归属汇总，合同在个人和团队视图中分别展示，但团队汇总只计算一次。

## 10. Excel 导入导出

- 支持客户、联系人、线索、商机和产品/服务目录模板导入。
- 导入流程为：上传 -> 解析 -> 全量校验 -> 预览 -> 用户确认 -> 执行。
- 导入批次具有唯一幂等标识，重复提交不会重复写入。
- 错误报告包含行号、字段、原值和错误原因。
- 用户可选择“全部通过才导入”或“导入正确行并保留错误报告”。
- 导出受当前数据范围和字段权限限制，并记录审计日志。

## 11. 数据一致性与异常处理

- 线索转化、报价转合同、合同生成回款计划等跨对象操作在单个数据库事务中完成。
- 客户、商机、报价和合同使用版本号实现乐观锁，冲突时提示用户刷新或对比差异。
- 关键对象采用状态、作废、更正和版本机制，不直接物理删除。
- 表单提交、Excel 导入和后台任务使用幂等键避免重复执行。
- 页面字段错误就地显示，并保留用户已输入内容。
- 客户重复、阶段门禁失败和权限不足提供明确原因及处理入口。
- 提醒和统计任务失败自动重试；持续失败时记录告警，不阻塞核心业务。

## 12. 安全、审计和非功能要求

### 12.1 安全

- Spring Security、强密码哈希、会话超时和登录失败限制。
- 全站 HTTPS；附件下载必须经过业务权限校验。
- 导入、导出、下载、权限变更和敏感数据访问留痕。
- 审计范围包括归属转移、阶段回退、金额与赢率变化、报价版本、合同状态、回款更正和附件删除。

### 12.2 性能与容量

- 面向 50-300 名用户和约 100 个并发会话。
- 常用列表、详情和保存接口目标为 `P95 < 2 秒`。
- 搜索、筛选、预测和报表查询建立针对性索引，并限制无界查询。
- 大批量导入、导出和报表生成采用后台任务并提供进度。

### 12.3 运维与恢复

- PostgreSQL 每日自动备份，并定期执行恢复演练。
- 附件存储与数据库备份保持可恢复的一致时间窗口。
- Docker 镜像版本化，数据库迁移可审计，发布支持回滚到上一应用版本。
- 应用提供健康检查、结构化日志和关键任务告警。

## 13. 测试策略

- 单元测试：阶段赢率、客户查重、金额、目标完成率、预测和权限规则。
- 模块集成测试：使用真实 PostgreSQL 验证事务、约束、查询和乐观锁。
- API 权限测试：覆盖本人、部门层级、显式协作、敏感字段和越权访问。
- 前端组件测试：表单校验、阶段门禁、权限态和复杂作战页组件。
- 端到端测试：线索转化、商机推进、报价签约、回款登记和 Excel 导入。
- 发布验证：数据库迁移、备份恢复、附件鉴权和核心接口性能测试。

## 14. 分阶段交付

1. 基础平台：工程骨架、组织用户、权限、审计、附件和部署。
2. CRM 主数据：线索、客户、联系人、关系网络、查重和 Excel 导入。
3. 商机与作战页：阶段门禁、团队、里程碑、决策链、销售地图、行动和预测。
4. 产品与报价：目录、自由项、报价版本和附件。
5. 合同与回款：合同、回款计划、实收、逾期和更正。
6. 经营工作台：目标、完成情况、漏斗、预测、重点项目和风险。
7. 移动适配、性能、安全和恢复验收。

每个阶段提供可运行、可测试的增量，后续阶段只能通过已定义的模块接口使用前序能力。

## 15. 验收标准

- 八类核心岗位只能访问授权功能、数据范围和字段。
- 客户重复创建被阻止，归属冲突可提交主管处理。
- 线索可完整转化为已有或新客户、联系人、商机和首次行动。
- 商机按确认的 `0% / 20% / 40% / 50% / 60% / 85% / 100%` 赢率计算预测。
- 项目作战页覆盖附件中的项目摘要、进度、决策、竞争、关键人、行动、成本和复盘。
- 报价版本不可覆盖历史，合同可生成多个回款节点并登记实收。
- 工作台正确显示年、季、月团队和个人签约/回款目标及完成情况。
- Excel 导入可校验、预览、防重复并输出逐行错误报告。
- 关键变更可追溯，越权访问被拒绝并记录。
- 核心端到端流程、恢复测试和性能基线全部通过。
